# SentinelAI Dashboards & AI Assistant

## Overview

The Dashboards & AI Assistant module comprises the primary visualization and user interaction layer of SentinelAI. It translates raw database records, JSON findings, and multi-hop graph structures into an intuitive, real-time visualization suite. This module empowers Security Operations Center (SOC) analysts and executives to visualize threat landscapes, explain complex alerts, and support rapid decision-making natively within the enterprise workflow.

## Responsibilities

* **Data Visualization:** Renders complex security intelligence and attack paths into interactive, comprehensible formats.
* **AI Interaction:** Provides an executive-friendly interface for conversational interaction with the AI Decision Intelligence Engine.
* **Alert Management:** Displays structured feeds of discrete heuristic alerts and compliance findings.

## Architecture

The frontend is built on Next.js leveraging the App Router in TypeScript. It utilizes a modern component-driven architecture, isolating feature logic, data fetching, and state management within dedicated directories.

## Workflow

### Execution Lifecycle

1. **Initialization:** An authenticated user accesses a protected dashboard route.
2. **Data Fetching & Visualization:** The layout initializes and retrieves normalized intelligence data from the backend API Gateway. Complex structures are parsed and visually rendered, such as mapping graph relationships into an interactive node map.
3. **Explainability Streaming:** Language model recommendations and context are streamed and formatted into an executive-friendly UI, directly leveraging the backend's Explainability Engine.
4. **Decision Support:** By synthesizing visual node graphs with conversational AI assistance, security personnel are equipped with the comprehensive context needed to execute immediate, supported security decisions.

## Features

* **AI Investigation Dashboard:** An interactive chat and reporting interface seamlessly integrated with the intelligence engines.
* **Attack Graph Visualization:** A highly interactive canvas allowing security engineers to visually explore the potential blast radius of a compromised identity.
* **Risk Findings Console:** A structured, filterable feed rendering the discrete heuristic alerts generated by the Risk Engine.
* **Audit Logs:** Fast, paginated data tables optimized for traditional manual log investigation and compliance auditing.

## Internal Components

* `(dashboard)`: The primary routing directory containing all authenticated application routes, natively segmented by feature.
* Feature Folders (`ai-investigation`, `attack-graph`, `risk-findings`): Encapsulate specific component logic and state management.

## Performance Considerations

* **Server-Side Rendering:** The application leverages Next.js Server Components to shift heavy data-fetching logic to the server, dramatically decreasing the client-side JavaScript bundle size.
* **Pre-compiled Assets:** Layout structures and global CSS are pre-compiled, ensuring near-instant page transitions and a highly responsive user experience across the dashboard ecosystem.
