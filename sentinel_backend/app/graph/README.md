# SentinelAI Knowledge Graph Builder

## Overview

The Enterprise Knowledge Graph Builder is responsible for constructing a dynamic "Digital Twin" of the organizational cloud environment in real-time. It enables rapid querying of multi-hop relationships and lateral movement, which traditional relational databases struggle to process efficiently.

## Responsibilities

* **Entity Modeling:** Transforms standardized enterprise security data into distinct graph nodes representing identities, resources, and origins.
* **Relationship Mapping:** Connects isolated entities through semantic relationships to model actual access patterns and behavioral flow.
* **Graph Maintenance:** Ensures graph updates are idempotent and secure against injection vulnerabilities.

## Architecture

The module communicates natively with a Neo4j graph database. Rather than storing cybersecurity data as isolated records in relational tables, this engine models every enterprise object as a graph node and maps the environment's topology to support advanced blast-radius calculations and attack path generation.

## Workflow

### Processing Flow

1. **Entity Merging:** Standardized enterprise entities generated by the Normalization Engine are converted into graph nodes. Idempotent merge operations are executed for identities, resources, and IP addresses.
2. **Semantic Linking:** Based on normalized log data, the engine dynamically links nodes, tracking interaction frequencies, initial occurrence timestamps, and most recent interaction times.
3. **Graph Publication:** Validated graph updates are committed, and the latest iteration of the Enterprise Knowledge Graph is made available to the Risk Intelligence Engine.

## Features

* **Identity Triangulation:** Links identity nodes to originating IP addresses to detect geographic anomalies and distributed access patterns.
* **Privilege Escalation Mapping:** Constructs relationship pathways to map identity chaining and role assumption.
* **Resource Access Graphs:** Builds edges representing resource access, enabling rapid blast-radius calculation.
* **Attack Path Generation:** Supports variable-length path queries to instantly visualize multi-stage exposures.

## Internal Components

* `graph_builder.py` (`GraphBuilder`): The primary abstraction layer wrapping the graph database transactions and orchestration logic.
* `neo4j_queries.py`: Contains the raw, parameterized Cypher statements utilized to securely build and query the graph architecture.

## Reliability & Resilience

* **Idempotent Execution:** Native merge statements guarantee that duplicate pipeline runs simply update timestamps and interaction counts, preventing topological corruption.
* **Cypher Injection Protection:** All queries utilize parameterized inputs, entirely mitigating Cypher injection vulnerabilities.
