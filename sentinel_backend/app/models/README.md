# SentinelAI Relational Data Models

## Overview

This module contains the SQLAlchemy Declarative models that define the schema for the PostgreSQL database. It serves as the authoritative source of truth for the relational portion of SentinelAI's hybrid data architecture.

## Responsibilities

* **Schema Definition:** Defines tables, columns, data types, and complex constraints.
* **Relationship Mapping:** Establishes foreign key relationships and ORM back-populates for seamless data traversal.
* **Data Integrity:** Enforces uniqueness, check constraints, and cascading deletes directly at the database level.

## Core Models

* **`MachineIdentity`:** The central entity representing a unique machine or user within a workspace.
* **`AccessLog`:** The time-series storage for raw security telemetry and API calls associated with identities.
* **`RiskScore` & `RiskFinding`:** Persists the aggregated heuristic penalties and human-readable deterministic evidence generated by the Risk Intelligence Engine.
* **`Integration`:** Stores the configuration state and securely encrypted credentials for active cloud integrations (e.g., AWS CloudTrail).

## Architecture Notes

* **Hybrid Alignment:** These relational models are synchronized with the Enterprise Knowledge Graph (Neo4j) to provide a complete digital twin of the environment.
* **Multi-Tenancy:** Models are designed to include `workspace_id` to enforce strict data isolation boundaries across tenants.
