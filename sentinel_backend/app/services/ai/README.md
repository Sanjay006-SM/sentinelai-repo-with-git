# SentinelAI Decision Intelligence & Explainability Engine

## Overview

This module provides the automated intelligence capability embedded within SentinelAI. It utilizes the Gemini 3.5 Flash Large Language Model to interpret, contextualize, and summarize complex security events for executive leadership and technical Security Operations Center (SOC) analysts.

## Responsibilities

* **Contextualization:** Automates the gathering of context across disparate tools, correlating raw logs with graph database relationships.
* **Insight Generation:** Converts verified risk evidence into meaningful insights, remediation recommendations, executive summaries, and compliance guidance.
* **Explainability:** Validates that every AI-generated statement is supported by underlying deterministic evidence from the Enterprise Knowledge Graph.

## Architecture

The module employs an orchestration-and-collection architecture, strictly isolating API interactions from internal data gathering. Crucially, the AI does not calculate risk; it solely interprets the verified risk evidence generated by upstream intelligence engines.

* **AI Decision Intelligence Engine:** The core reasoning layer that wraps the language model SDK, managing retries, payload configuration, and output parsing via Retrieval-Augmented Generation (RAG).
* **Explainability Engine:** Serves as the trust layer, retrieving and attaching exact evidence to prevent hallucination and validate claims.

## Workflow

### Processing Flow

1. **Evidence Collection:** A multi-dimensional snapshot of the target identity is extracted from PostgreSQL and Neo4j. This forms the verifiable truth that grounds the AI reasoning.
2. **Prompt Optimization:** Collected evidence is formatted into a highly constrained prompt designed to enforce strict JSON output schemas, optimizing inference latency and remaining within context window limits.
3. **LLM Inference:** The formulated prompt is submitted to the language model. The request strictly defines the required response MIME type to guarantee structured output.
4. **Validation & Parsing:** The raw string output is parsed and statically validated against a required schema to guarantee API contract stability before passing the payload upstream to the API gateway.

## Features

* **Constrained JSON Prompting:** Guarantees the programmatic usability of the LLM output and prevents downstream crashes due to missing nodes or malformed structures.
* **Automated Context Gathering:** Eliminates the manual correlation phase typically required by SOC analysts.
* **Verifiable Explanations:** Employs explicit prompt directives requiring all findings to be strictly based on provided evidence, eliminating assumptions regarding permissions.

## Internal Components

* `ai_analyst_service.py` (`AIAnalystService`): Manages the language model lifecycle, inference execution, and schema parsing.
* `prompt_builder.py` (`PromptBuilder`): Compiles telemetry into constrained prompt templates.
* `evidence_collector.py` (`EvidenceCollector`): Retrieves and synthesizes verifiable evidence from the persistence layers.

## Integration

The module acts as a critical intermediary, ingesting deterministic risk intelligence from the Risk Intelligence Engine (Stage 4) and delivering formatted, human-readable insights to the API Gateway (Stage 7).

## Performance Considerations

* **Database Aggregation:** To optimize token usage, heavy SQL aggregation is performed locally. The system feeds synthesized integers and summaries into the language model rather than thousands of raw log entries.
* **Token Guardrails:** Recent activity logs are explicitly limited to prevent context window bloat and control inference costs.

## Error Handling

* **Inference Resiliency:** Language model network calls are protected by automatic retries and exponential backoff strategies to gracefully handle transient API instability.
* **Schema Fallbacks:** Missing string or array keys in the parsed output are safely initialized to empty defaults, maintaining the stability of downstream contracts.
